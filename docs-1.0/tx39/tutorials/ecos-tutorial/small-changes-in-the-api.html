Small changes in the API

Since the previous release in June 1998, some functions in the C kernel API have changed, mostly in the order in which they take their arguments.

  • Almost all "with timeout" system calls that used to use relative timeout times now use absolute times.

  • The order of stack_base and stack_size in cyg_thread_create() has been swapped.

  • The type cyg_addrword is now called cyg_addrword_t.

  • Many system calls that used to use void * as a pointer to data now use cyg_addrword_t.

  • A new entry point for C programs has been introduced: the function cyg_user_start(). The chapter System startup in the eCos Reference Manual contains a complete description of how this works. If main() is still used as an entry point, it will run in its own thread.
