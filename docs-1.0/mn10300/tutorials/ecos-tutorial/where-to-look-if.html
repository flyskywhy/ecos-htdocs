Chapter 10. Where to look if…

Now we will step through a list of needs you might have when programming eCos, and we will point you to the various parts of the documentation that might help you find answers.

If you want to

install eCos

See Release Notes and Installation Guide.

program with eCos

Use the programming tutorial (see Programming Tutorial) to learn to write and run simple eCos programs. After that, most of the reference manual is appropriate reading.

configure eCos for your purposes

  • You can learn about the graphical front end to configuration by reading Chapter 14, and the part eCos User Guides called The eCos Configuration Tool.

  • You can also learn about the underlying engine, called pkgconf.tcl, in the chapter Invoking pkgconf.tcl from the command line, as well as the Reference for pkgconf.tcl, in the eCos User Guides.

see what has changed in the kernel API (other than the addition of new functionality)

Read the section called Small changes in the API in Chapter 5

find details about the configuration tool

Read the part of the eCos User Guides called The eCos Configuration Tool.

get the whole scoop on any configuration option

Read the part of the eCos User Guides called Configuration Option Reference.

write or port a µITRON application

For now, the only information is in the µITRON API chapter in the eCos Reference Manual, as well as Dr. Sakamura's book on µITRON which we ship with this release. The next release of eCos will include a tutorial on writing and porting µITRON programs.

run an existing eCos test

See Chapter 14.

reduce code size

Code size is already greatly reduced using gcc's selective linking, but you can reduce it further by configuring the kernel and the C library to use different implementations of certain features and to eliminate unneeded functionality.

import a new eCos release into your own source control system

Read the chapter Managing the package repository in the eCos User Guides.

port eCos to a new platform

Read the discussion of platform porting in the section Kernel porting notes in the appendix The eCos Hardware Abstraction Layer (HAL) in the eCos Reference Manual.

write a new device driver

The device driver infrastructure will be changing in the next release, but some information can be found in the eCos Device Drivers part of the eCos Reference Manual.
