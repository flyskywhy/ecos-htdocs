Chapter 6. A note on testing

The eCos kernel and other packages have test suites which rigorously exercise the available features and confirm correct execution. The use of test suites is particularly important for embedded systems, where software robustness is a priority. All eCos software is tested prior to shipping, but if you define your own configuration, you will probably want to verify that the test cases work for that particular configuration.

This release includes test suites for the eCos kernel, kernel C API, C library and µITRON compatibility packages. The use of the test suites is similar for all packages. The tests are supplied both as prebuilt images and as source code for building with specific eCos configurations.

Each test suite consists of a number of test cases which must be executed individually by following these steps:

  1. The test cases are compiled and linked with the appropriatly configured eCos packages, using the make tests command (further details in Chapter 14).

  2. The baud rate is set up, usually with (gdb) set remotebaud 38400.

  3. The target board is selected, usually with (gdb) target remote com1.

  4. The test program is downloaded, usually with (gdb) load. This can take some time.

  5. The test program is executed, usually with (gdb) continue.

Each test case runs without further intervention. A test case may involve one or more individual tests. Successful completion of each test within the test case is reported as a line of text which is sent to the diagnostic channel (usually the serial port) for display on a terminal or terminal emulator.

Each test case runs only once and usually requires the target hardware to be reset on completion. Note that certain test cases may not terminate immediately, especially if they involve delays and run on the target simulators.

To see which test cases are provided, you can examine the following directories (assuming you installed eCos in the default location):

  • prebuilt tests compiled for RAM startup (need CygMon, and will only run on real hardware) are located in C:\Program Files\Cygnus Solutions\eCos\prebuilt\v1_0\mn10300-stdeval-ram

  • prebuilt tests compiled for ROM startup (will work in real hardware and in the simulator) are located in C:\Program Files\Cygnus Solutions\eCos\prebuilt\v1_0\mn10300-stdeval-ram

  • test case source code is under the base source directory C:\Program Files\Cygnus Solutions\eCos\packages\

    • compat\uitron\v1_0\tests

    • devs\serial\rs232\common\v1_0\tests

    • devs\serial\rs232\mn10300\v1_0\tests

    • devs\wallclock\v1_0\tests

    • devs\watchdog\v1_0\tests

    • kernel\v1_0\tests

    • language\c\libc\v1_0\tests

    • language\c\libm\v1_0\tests

In the section called Verification in Chapter 12 you will find detailed instructions on running a test case to verify that your hardware board is working with the supplied software.
