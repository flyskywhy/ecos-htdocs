Chapter 13. Introduction: the quickest way to start programming eCos

For many people the easiest way to start using a new software system is to run through a simple and yet complete example of its use.

This informal tutorial will take you through the most straightforward path to build and run your own eCos application.

We assume that you have installed eCos and Cygnus's GNUPro suite of tools, which are usually installed with eCos. This also installs the Cygwin32 POSIX environment for NT.

We will be using the bash command line interpreter that comes with Cygwin32. Keep in mind that bash uses UNIX-style path names, so that instead of C:\Program Files\Cygnus Solutions\eCos we will use //c/Program\ Files/Cygnus\ Solutions/eCos in this tutorial.

We will assume that you installed eCos in //c/Program\ Files/Cygnus\ Solutions/eCos, and that you will build eCos in //c/ecos-work. If you are using different directories, please change the paths in the examples below.

The steps we will go through in this tutorial are:

  1. Configure the eCos kernel for your particular platform.

  2. Build that kernel with its associated libraries and test suites.

  3. Run one of the test programs that are shipped with eCos (built in the previous step).

  4. Build some eCos applications of our own, starting with a trivial hello world style of program, then moving on to a multi–threaded program, and finally to a program that uses some of the eCos system calls to do (almost) useful work.
