Installing eCos under Linux

Downloading

The eCos 1.2.1 distribution for Linux is available as an archive file in both bzip2 (.tar.bz2) and gzip (.tar.gz) formats. Click on one of the following links to download:
  • eCos 1.2.1 for Linux (ecos-1.2.1.tar.bz2 - 2MB)
  • eCos 1.2.1 for Linux (ecos-1.2.1.tar.gz - 3MB)
In order to build the eCos sources, you will require a cross-compiler and related development tools. The necessary tools are designated ecosSWtools-arm-990321 for the ARM target and ecosSWtools-990319 for the MN10300, PowerPC, SPARClite and TX39 targets. They are available in source form only. Click on one of the following links to download:
  • eCos development tools for MN10300, PowerPC, SPARClite and TX39 (ecosSWtools-990319-src.tar.bz2 - 23MB)
  • eCos development tools for MN10300, PowerPC, SPARClite and TX39 (ecosSWtools-990319-src.tar.gz - 28MB)
  • eCos development tools for ARM (ecosSWtools-arm-990321-src.tar.bz2 - 17MB)
  • eCos development tools for ARM (ecosSWtools-arm-990321-src.tar.gz - 21MB)

Users of Redhat Linux 6.0 should look at the FAQ entry for toolchain build problems under Linux as a patch is required for these systems.

Note that Linux users may optionally build eCos for the synthetic Linux target using egcs version 1.1.1 or later. The Hardware setup chapter in the eCos documentation set provides further detail.

Installation

  1. If you have . in your PATH environment variable, make sure that it comes after the GNU native compiler tools directory. The following build procedure will fail if . is ahead of the GNU native compiler tools in the path.

  2. Build the eCos development tools by following these steps at a sh, ksh or bash prompt. Users of the csh and tcsh shells should replace "> filename.out 2>&1" with ">& filename.out" throughout:

    • Extract the eCos development tool sources from the downloaded distribution file:

      mkdir /usr/cygnus
      cd /usr/cygnus
      bunzip2 < ecosSWtools-arm-990321-src.tar.bz2 | tar xvf -           (for the ecosSWtools-arm-990321 bzip2 archive)
      gunzip < ecosSWtools-arm-990321-src.tar.gz | tar xvf -       (for the ecosSWtools-arm-990321 gzip archive)
      bunzip2 < ecosSWtools-990319-src.tar.bz2 | tar xvf -         (for the ecosSWtools-990319 bzip2 archive)
      gunzip < ecosSWtools-990319-src.tar.gz | tar xvf -           (for the ecosSWtools-990319 gzip archive)

      The directory /usr/cygnus/ecosSWtools-arm-990321/src or /usr/cygnus/ecosSWtools-990319/src should be generated and populated during the extraction process.

    • Create an object directory for your builds:

      mkdir /tmp/build
      cd /tmp/build

    • Configure the tools, saving the output to configure.out:

      /usr/cygnus/ecosSWtools-arm-990321/src/configure -v \
        --prefix=/usr/cygnus/ecosSWtools-arm-990321 \
        --exec-prefix=/usr/cygnus/ecosSWtools-arm-990321/H-host-triplet \
        --target=arm-elf > configure.out 2>&1                                      (for ecosSWtools-arm-990321)

      /usr/cygnus/ecosSWtools-990319/src/configure -v \
        --prefix=/usr/cygnus/ecosSWtools-990319 \
        --exec-prefix=/usr/cygnus/ecosSWtools-990319/H-host-triplet \
        --target=target-triplet > configure.out 2>&1                               (for ecosSWtools-990319)

      You should replace target-triplet in the above command with one of the following target triplets: i686-pc-linux-gnu, mn10300-elf, powerpc-eabi, sparclite-elf or mips-tx39-elf. You should replace host-triplet with the host triplet for your build system. You can determine the host triplet by running /usr/cygnus/ecosSWtools-arm-990321/src/config.guess or /usr/cygnus/ecosSWtools-990319/src/config.guess.

    • Build and install the tools, saving the output to make.out:

      make -w all install LIBGCC2_DEBUG_CFLAGS="-g1 -O1" \
        LANGUAGES="c c++" > make.out 2>&1                            (for ecosSWtools-arm-990321)

      make -w all install LANGUAGES="c c++" > make.out 2>&1    (for ecosSWtools-990319)

    When the build is complete, the executable files of the eCos development tools may be found in the directory /usr/cygnus/ecosSWtools-arm-990321/H-host-triplet/bin or /usr/cygnus/ecosSWtools-990319/H-host-triplet/bin. You should add this directory to the head of your PATH environment variable.

  3. Extract the eCos repository using the following commands:

    cd /usr/local
    bunzip2 < ecos-1.2.1.tar.bz2 | tar xvf -         (for the bzip2 archive)
    gunzip < ecos-1.2.1.tar.gz | tar xvf -       (for the gzip archive)

    On completion, the eCos repository may be found in the directory /usr/local/ecos-1.2.1.
