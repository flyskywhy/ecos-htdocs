eCos Sourceware

Getting Started

Downloading

The eCos sourceware distribution is available in both Linux and Windows versions. The Linux version has been tested under RedHat Linux 5.1 but should work under most Linux variants. The Windows version has been tested under Microsoft Windows NT 4.0 and also works under Windows 95 and Windows 98. Click on one of the following links to download:
  • eCos 1.1 sourceware for Linux (4MB tar.bz2 file)
  • eCos 1.1 sourceware for Windows (10MB self-extracting executable file)
In order to build the eCos sources, you will require the a cross-compiler and related development tools. The necessary tools are designated ecosSWtools-981021 and are available in source form only. Note that, although eCos may be configured and built under Windows 95 and Windows 98, it is not possible to build the eCos sourceware development tools on these platforms. If you wish to develop eCos applications using Windows 95 or Windows 98, you must build the development tools under Windows NT. Click on the following link to download:
  • eCos sourceware development tools (33MB zip file)
The eCos development tools make use of GNU native tools on both Linux and Windows. Linux users will already have the necessary tools. Windows NT users should use the GNU-Win32 Beta 20 native tools which are available in pre-built form. Click on the following link to download from one the GNU-Win32 mirror sites:
  • GNU-Win32 Beta 20 development tools

Installing

Installing under Windows

  1. Install the GNU-Win32 native tools by running the downloaded distribution file and following the on-screen instructions. The remaining instructions assume that the GNU-Win32 tools are installed to the default location. If you wish to install to a different location, you should avoid the use of spaces in the destination path. Once installed, select Cygwin32 Beta 20 from the start menu hierarchy to launch the bash command-line environment.

  2. Build the eCos sourceware development tools (ecosSWtools-981021) by following these steps at the bash prompt:

    • Mount the logical drive(s) you will use to build the eCos sourceware development tools:

      mount C: /c

    • Mount the GNU-Win32 tools as /bin such that /bin/sh exists:

      mount C:/cygnus/b20/H-i586-cygwin32/bin /bin

    • Mount the GNU-Win32 libraries as /lib:

      mount C:/cygnus/b20/H-i586-cygwin32/i586-cygwin32/lib /lib

    • Set the MAKE_MODE environment variable to "unix":

      export MAKE_MODE=unix

    • Extract the eCos sourceware development tool sources from the downloaded distribution file to C:\cygnus using a utility such as WinZip. You should ensure that the utility is configured to preserve the directory structure within the zip file during extraction. The directory C:\cygnus\ecosSWtools-981021\src should be generated and populated during the extraction process.

    • Create an object directory for your builds:

      mkdir /c/build
      cd /c/build

    • Configure the tools, saving the output to configure.out:

      /c/cygnus/ecosSWtools-981021/src/configure -v \
        --prefix=/c/cygnus/ecosSWtools-981021 \
        --exec-prefix=/c/cygnus/ecosSWtools-981021/H-i586-cygwin32 \
        --host=i586-cygwin32 --target=target-triplet > configure.out 2>&1

      You should replace target-triplet in the above command with one of the supported target triplets: mn10300-elf, mips-tx39-elf or powerpc-eabi.

    • Build and install the tools, saving the output to make.out:

      make -w all install > make.out 2>&1

    When the build is complete, the executable files of the eCos sourceware development tools may be found in the directory C:\cygnus\ecosSWtools-981021\H-i586-cygwin32\bin.

  3. Install eCos by running the downloaded distribution file and following the on-screen instructions.

Installing under Linux

  1. Build the eCos sourceware development tools (ecosSWtools-981021) by following these steps at the bash prompt:

    • Extract the eCos sourceware development tool sources from the downloaded distribution file to /usr/cygnus using the unzip utility or similar. You should ensure that the utility is configured to preserve the directory structure within the zip file during extraction. The directory /usr/cygnus/ecosSWtools-981021/src should be generated and populated during the extraction process.

    • Create an object directory for your builds:

      mkdir /tmp/build
      cd /tmp/build

    • Configure the tools, saving the output to configure.out:

      /usr/cygnus/ecosSWtools-981021/src/configure -v \
        --prefix=/usr/cygnus/ecosSWtools-981021 \
        --exec-prefix=/usr/cygnus/ecosSWtools-981021/H-host-triplet \
        --target=target-triplet > configure.out 2>&1

      You should replace target-triplet in the above command with one of the supported target triplets: mn10300-elf, mips-tx39-elf or powerpc-eabi. You should replace host-triplet with the host triplet for your build system. You can determine the host triplet by running /usr/cygnus/ecosSWtools-981021/src/config.guess.

    • Build and install the tools, saving the output to make.out:

      make -w all install > make.out 2>&1

    When the build is complete, the executable files of the eCos sourceware development tools may be found in the directory /usr/cygnus/ecosSWtools-981021/H-host-triplet/bin. You should add this directory to your path.

  2. Extract the eCos repository using the following commands:

    cd /usr/local
    bunzip2 < ecos-1.1.tar.bz2 | tar xvf -

    On completion, the eCos repository may be found in the directory /usr/local/ecos-1.1.
