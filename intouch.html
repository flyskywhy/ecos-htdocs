Keeping in Touch

Mailing lists

There are two principal mailing lists relating to eCos. Subscribers to the ecos-announce list will receive details of all eCos developments as they unfold. This is a moderated list intended principally for the announcement of new packages, snapshots and patches. Subscribers to the ecos-discuss list can participate in open discussion with others concerning all aspects of eCos. This is an unmoderated list. Archives of both the ecos-announce and ecos-discuss mailing lists are available.

We strongly recommend you read both the eCos documentation fully and search the above mail archives before asking simple questions on the discussion list, particularly if they are likely to be problems that people would have encountered before.

There are additional mailing lists specifically for those requiring notification of changes at this site. Subscribers to the ecos-cvs list will receive notification of any changes to the eCos anonymous CVS repository. Archives of ecos-cvs are also available. Subscribers to the ecos-webpages-cvs list will receive notification of any changes to the eCos web pages.

To subscribe to any of these mailing lists, use the mailing list request form below. Digest versions of each list are also available where messages are forwarded on a daily basis. If you are experiencing problems with the use of these mailing lists, please refer to the mailing list FAQ.

Contacting Red Hat

Commercial enquires covering eCos development, porting and support contracts should be addressed to the eCos commercial team at <ecos-info@redhat.com>. Private technical queries concerning eCos development and collaboration should be addressed to the eCos maintainers <ecos-maintainers@redhat.com>. Please note that the eCos maintainers are unable to provide technical support, and general technical queries or requests for help or information that would have been more appropriate for <ecos-discuss@sources.redhat.com> may not be answered.

Please use our pre-sale inquiries address <presales@redhat.com> for information concerning other Red Hat products.
