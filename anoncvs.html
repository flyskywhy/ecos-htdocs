Anonymous CVS Access

Introduction

The most recent versions of eCos and RedBoot may be obtained using anonymous CVS access. This facility provides immediate access to the latest eCos and RedBoot features and the rapid deployment of any necessary bug fixes.

For those unfamiliar with CVS, the following resources may be of use:

  • CVShome.org has CVS clients available for download. Other information and links to books on the subject can also be found at this site.
  • A good starting point is the Introduction to CVS by Jim Blandy.
  • The CVS sources come with texinfo documentation which is also available in HTML format.
We also provide a simple but effective way to view the CVS repository over the web using CVSweb. You can inspect the contents, history and differences between versions of all files in the repository in this way.

The CVS source repository

The anonymous CVS repository contains the latest version of our master internal repository and is generally updated on a weekly basis. However, users should note that these updates have not been rigorously tested and cannot be guaranteed to build or function correctly. It is therefore recommended that new developers work with the latest full release of eCos and RedBoot which have been put through extensive testing and QA. The repository should be used in situations where you need access to new features, bug fixes, or are working on a shared contributed project.

The snapshots include as much source code as we are able to publicize. However, there may be new architectural ports, board ports, or areas of functionality which are not released for commercial or contractual reasons.

The relevant host side build tools may be obtained by FTP from this site and are not provided by anonymous CVS. Any patches or changes required to enable the use of these tools with the latest version of eCos in the repository will be publicized. If any updates to the eCos Configuration Tool are required, we will put up a new binary as soon as possible. You should ensure that you use the latest version of the Configuration Tool from the most recent net release of eCos when working with the sources from the anonymous CVS repository.

Accessing the repository

Before you can access the CVS repository, you must have a CVS client (see above) and direct TCP/IP access. If you are behind a firewall, you should discuss access with your local systems administrator.

Type the following command to login to the CVS server:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos login

You will be prompted for a password which is anoncvs. You only need to perform this login the first time you access the eCos CVS repository. The password is saved in the file .cvspass in your home directory for use in subsequent sessions.

It is recommended that you use different directory trees for the anonymous CVS sources and for full releases of eCos. Create a new directory for the check out, change to that directory, and type:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos -z 6 co -P ecos

This command should initiate a connection to the CVS server and check out the latest version of eCos. In the future, when you want to update your repository to the latest version, go to the base of the repository tree and use the command:

cvs -z 6 update -d -P

Using the repository

Once the sources have been checked out, Unix users (or Windows users using the command line) can switch to it by setting the ECOS_REPOSITORY environment variable to the ecos/packages subdirectory of the checked out tree. For example:

ECOS_REPOSITORY=/ecoscvs/ecos/packages ; export ECOS_REPOSITORY (for sh, ksh and bash users)
setenv ECOS_REPOSITORY /ecoscvs/ecos/packages (for csh and tcsh users)

Users of the eCos Configuration Tool should select the new repository using the Build->Repository menu item.

Use of a new repository with an old build tree is not recommended. Changes may have been made to the repository CDL files that will not be present in the working versions in your build tree.

Optional networking support

Networking support is an optional feature in eCos and is not checked out from the repository by default. The 1.3.1 EPKs are not compatible with the code in the anonymous CVS repository. Such support can be added to an already checked out tree by executing the following command:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos -z 6 co -P net

A port of the University of California at Davis (UCD) SNMP stack is also available as a separate optional package, and may be checked out by additionally executing the following command:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos -z 6 co -P snmp

Note that the networking and SNMP package records will be present in the package database (ecos.db) regardless of whether networking or SNMP support is checked out. Updated configuration tools (see below) are required when the networking/SNMP support is not present. The eCos 1.3.1 tools will report errors.

Release tags

Our public releases have all been tagged in the repository so that you can check out the official release of any particular version. More usefully, these tags can be used to view the differences between e.g. the last officially released version and the latest version in the repository:

cvs diff -r ecos-v1_1-release ecos/packages/targets

The release tags all take the form ecos-<version>-release (eg ecos-v1_3_1-release). The repository is also tagged every time a new snapshot is generated. These tags are of the form ecos-sw-<date in ISO format> (eg ecos-sw-1999-05-11).

To check out a particular version from scratch, use a command such as:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos co -r ecos-v1_3_1-release ecos

As with all such sticky checkouts, a cvs update will not update your sources to the latest version.

Host configuration tools

The current set of sources in the CVS repository require an updated version of the host-side configuration tools. You may wish to recompile these yourself using the host tool sources available from anonymous CVS. Alternatively, unsupported prebuilt binaries are available for Windows and Red Hat Linux 6.x hosts.

Updated configuration tools for Windows

Self-extracting zip files for updated binaries of both the eCos Configuration Tool for Windows and the command-line ecosconfig tool are available from the eCos FTP site or preferably a mirror site.

It is recommended that you rename the existing files Configtool.exe and ecosconfig.exe to Configtool.exe.old and ecosconfig.exe.old before proceeding. These files are found in the tools/bin subdirectory of your eCos installation.

Install the revised configuration tools by executing the downloaded files. Modify the suggested target folder as necessary to place the extracted files in the tools/bin directory of your eCos installation.

To update the platforms list for the "Run Tests" feature, you will need to download and run this registry fragment. Once downloaded, simply double click the file to install it.

Note that using the updated Configuration Tool with an eCos v1.3.1 repository will lead to problems if the memory layout is modified.

Updated configuration tools for Red Hat Linux 6.x and 7.x

An unsupported prebuilt binary of the updated ecosconfig tool for Red Hat Linux 6.x and above is available for download from the eCos FTP site or preferably a mirror site.

It is recommended that you rename the existing ecosconfig executable to ecosconfig.old before proceeding. This file is located in the tools/bin subdirectory of your eCos installation.

Uncompress the updated tool with the commands:

gzip -N -d ecosconfig1314-linux.gz
chmod +x ecosconfig

Move the resulting ecosconfig executable into the tools/bin subdirectory of your eCos installation.

Other notes

Anonymous CVS access to the repository is read-only. If you would like to contribute a change, bug fix or other improvement to eCos then you should start by discussing it on the ecos-discuss mailing list. If applicable, you may wish to post (or put up for FTP) the relevant diff or patch.

You can obtain e-mail notifications when changes are made to the eCos repository. Refer to our keeping in touch page for further details.

Since the anonymous CVS repository is copied from the master repository, the author field and log entries are not accurate. Use the ChangeLog files at the top of each package hierarchy to obtain history on the sources in that package.

David Airlie of Parthus Technologies plc. has announced the availability of FTP snapshots. It is possible to obtain complete snaphosts and individual files. This service may be particularly useful for people unable to run CVS directly, for example because of corporate firewalls.

Queries and problems regarding use of the anonymous CVS service should be directed to the ecos-discuss mailing list. The eCos development team, including the anonymous CVS maintainer, read this list. Please clarify that the mail concerns an anonymous CVS problem in the subject field.
