Contributions

Introduction

Contributions to eCos are always welcome. These may be in the form of a new package, enhancements to an existing package or bug fixes. A contribution may be from a individual or part of a collaborative project. Information on how to contribute is contained within the eCos FAQ.

External projects

The following external projects are known to the eCos maintainers. If you are working on an eCos-based project that others could benefit from or contribute to, please let the eCos maintainers know and it will be added to this page. Sources are provided as is and are not supported by Red Hat at this time.

TCP/IP stack

Tristan Gingold has ported RTEMS libnetworking to eCos. This includes a FreeBSD-derived TCP/IP stack. The sources are available for download at:

ftp://sourceware.cygnus.com/pub/ecos/contrib/io/tcpip/rtems/

Note that this contribution is not related to the eCos TCP/IP stack.

PC platform HAL

Patrick O'Grady has developed an eCos HAL for the PC platform. A GDB stub is provided, allowing the developer to download applications to the target PC via a serial port. The sources are included in the eCos net distribution as of release 1.3.1.

Synthetic Linux platform HAL

Chris Provenzano has developed the eCos i386/Linux synthetic target. This is essentially a port of the eCos HAL to a Linux process and allows the developer to run eCos applications under Linux as if they were native Linux applications. The sources are included in the eCos net distribution as of release 1.2.1.

Kaffe Java VM

Ryouzaburou Suzuki has ported the Kaffe Java VM to eCos 1.3.1 on SH4. It is available for download here. Note that it is covered by the GNU General Public License.

Floating point hardware support

Sergei Organov is working on support for floating point hardware within eCos. He is targeting the PowerPC processors initially. Sources are not yet available.

NEC V850 architecture HAL

Takayuki Wakabayashi has developed an eCos HAL for the NEC V850 architecture. The port was developed using the NEC V850 simulator and has not as yet been targeted at a specific hardware platform. The port is completely unsupported and has not been tested at Red Hat. The sources will be available shortly.

Motorola MPC8xxFADS platform HAL

Kevin Hester has developed an eCos HAL for the Motorola MPC8xx Family Application Development System platform. The sources are included in the eCos net distribution as of release 1.2.1.

Samsung SNDS platform HAL

Grant Edwards is working on a port to the Samsung KS32C5000 microcontroller (ARM7TDMI core) and SNDS development hardware. Sources are not yet available (estimated availability - January 2000).

ARM Integrator HAL and drivers

Philippe Robin at ARM ARM Integrator platform. This port is available in tarred gzip format here.

It is supplied as-is and was made against anonymous CVS as of the beginning of October 2000. It will inevitably require some merging to work against current anonymous CVS sources. There are some more comments from Philippe on the status as contributed in this readme.

When the port is finished, it will be integrated into the anonymous CVS repository in due course.

CarbonKernel RTOS simulator support

Realiant Systems has made available an eCos personality for the CarbonKernel RTOS simulator. CarbonKernel is an event-driven simulator which mimics RTOS behaviour, allowing development and testing of embedded software on a workstation. More details can be found at the CarbonKernel development site.

Bluetooth and WAP stack

Andreas Karlsson at Combitech Systems AB has made available a port of the Axis Bluetooth stack. This was ported to develop an embedded WAP server running on top of Bluetooth.

The source code is available AS-IS from Combitech's web site .

Note that this stack is covered by the GNU General Public License.

x86 NE2000 ethernet driver

Christian Plessl and Thomas Meyer have ported an Ne2000 driver from OpenBLT to eCos. It is restricted to the x86 PC port only. The README contains details and instructions from Christian's announcement to the ecos-discuss list. The driver package tarball and addto.ecos.db file can be downloaded here.
