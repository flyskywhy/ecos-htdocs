Introduction

What is it?

eCos is an infrastructure for developing embedded applications. It consists of:

  • A configuration system which presents eCos as a component architecture, and allows fine-grained configuration of each component. Using the configuration system you can compose your own operating system and fine-tune it to do what your application needs.
  • Host-side libraries and tools to support this configuration infrastructure.
  • A real time kernel.
  • An ISO standard C and math library.
  • An implementation of the µITRON operating system, running on top of the eCos kernel.
  • A collection of device drivers (not too many so far).
  • GDB and simulator support for the target CPUs, so that you can run and debug both in simulation and on real hardware.

eCos's kernel and runtime components are open source software, and we encourage you to participate in the development of eCos.

For a description of our goals, please read the Foreword in the eCos online documentation.

What platforms are supported?

Target architectures

At the time I write this, 1998-10-28, the following CPUs are supported:

  • Matsushita MN10300
  • Toshiba TX39
  • PowerPC

Host operating systems

At the time I write this, 1998-10-28, the following host operating systems are supported:

  • Microsoft Windows NT 4.0, Windows 98, Windows 95
  • UNIX (Linux in particular)

Compilers, debuggers and simulators

ecos requires some new features from the GNU compiler toolchain, and we have made these new versions available here to accompany this release.

The version of GDB provided here has support for the MN10300, TX39 and PowerPC simulators.
