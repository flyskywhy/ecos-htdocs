Anonymous CVS Access

Introduction

The most recent versions of RedBoot may be obtained using anonymous CVS access. This facility provides immediate access to the latest RedBoot features and the rapid deployment of any necessary bug fixes.

For those unfamiliar with CVS, the following resources may be of use:

  • CVShome.org has CVS clients available for download. Other information and links to books on the subject can also be found at this site.
  • A good starting point is the Introduction to CVS by Jim Blandy.
  • The CVS sources come with texinfo documentation which is also available in HTML format.
We also provide a simple but effective way to view the eCos and RedBoot CVS repository over the web using CVSweb. You can inspect the contents, history and differences between versions of all files in the repository in this way.

The RedBoot repository

The CVS source repository used is the same as the eCos anonymous CVS repository. This is because RedBoot is based on eCos. For that reason, if you already have an existing checked out eCos source tree, you do not need to check out RedBoot sources separately. The RedBoot (and eCos) anonymous CVS repository contains the latest version of our master internal repository and is generally updated on a weekly basis. However, users should note that these updates have not been rigorously tested and cannot be guaranteed to build or function correctly. If you have a stable and tested release for your platform available to you, in general that should be used in preference. The repository should be used in situations where you need access to new features, bug fixes, or are working on a shared contributed project.

The snapshots include as much source code as we are able to publicize. However, there may be new architectural ports, board ports, or areas of functionality which are not released for commercial or contractual reasons.

The relevant host side build tools may be obtained by FTP from this site and are not provided by anonymous CVS. Any patches or changes required to enable the use of these tools with the latest version of eCos in the repository will be publicized. If any updates to the eCos Configuration Tool are required, we will put up a new binary as soon as possible. You should ensure that you use the latest version of the eCos Configuration Tools which may be downloaded below when working with the sources from the anonymous CVS repository.

Accessing the repository

Before you can access the CVS repository, you must have a CVS client (see above) and direct TCP/IP access. If you are behind a firewall, you should discuss access with your local systems administrator.

Type the following command to login to the CVS server:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos login

You will be prompted for a password which is anoncvs. You only need to perform this login the first time you access the eCos/RedBoot CVS repository. The password is saved in the file .cvspass in your home directory for use in subsequent sessions.

If you have an existing eCos installation, it is recommended that you use different directory trees for the anonymous CVS sources and for full releases of eCos.

To check out a complete set of sources for all targets, Create a new directory for the check out, change to that directory, and type:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/ecos -z 9 co -P redboot-all

This command should initiate a connection to the CVS server and check out the latest version of RedBoot. In the future, when you want to update your repository to the latest version, go to this same directory, and repeat the above command.

Users who have used cvs before may be tempted to perform a cvs update -d operation, however if this is used at the top level, this will check out a full eCos source repository, containing many more files than are required to build just RedBoot.

Downloading sources for particular targets

When executing the above command, the checked out sources include all ports and device drivers, including even those on which RedBoot is not supported (but which are supported by eCos). For some targets, we are available to provide special modules which contain only the sources required to build RedBoot on that target.

At this time, the only target supported with this mechanism is the ipaq. To check out ipaq-only RedBoot sources, use the checkout command as above, but replace redboot-all with redboot-ipaq.

Using the repository

Once the sources have been checked out, Unix users (or Windows users using the command line) can switch to it by setting the ECOS_REPOSITORY environment variable to the ecos/packages subdirectory of the checked out tree. For example:

ECOS_REPOSITORY=/ecoscvs/ecos/packages ; export ECOS_REPOSITORY (for sh, ksh and bash users)
setenv ECOS_REPOSITORY /ecoscvs/ecos/packages (for csh and tcsh users)

Users of the eCos Configuration Tool should select the new repository using the Build->Repository menu item.

Use of a new repository with an old build tree is not recommended. Changes may have been made to the repository CDL files that will not be present in the working versions in your build tree.

Host configuration tools

The current set of sources in the CVS repository require an updated version of the host-side configuration tools. You may wish to recompile these yourself using the host tool sources available from anonymous CVS. Alternatively, unsupported prebuilt binaries are available for Windows and Red Hat Linux 6.x hosts.

Updated configuration tools for Windows

Self-extracting zip files for updated binaries of both the eCos Configuration Tool for Windows and the command-line ecosconfig tool are available.

It is recommended that you rename the existing files Configtool.exe and ecosconfig.exe to Configtool.exe.old and ecosconfig.exe.old before proceeding. These files are found in the tools/bin subdirectory of your eCos installation.

Install the revised configuration tools by executing the downloaded files. Modify the suggested target folder as necessary to place the extracted files in the tools/bin directory of your eCos installation.

To update the platforms list for the "Run Tests" feature, you will need to download and run this registry fragment. Once downloaded, simply double click the file to install it.

Note that using the updated Configuration Tool with an eCos v1.3.1 repository will lead to problems if the memory layout is modified.

Updated configuration tools for Red Hat Linux 6.x and 7

An unsupported prebuilt binary of the updated ecosconfig tool for Red Hat Linux 6.x and above is available for download.

It is recommended that you rename the existing ecosconfig executable to ecosconfig.old before proceeding. This file is located in the tools/bin subdirectory of your eCos installation.

Uncompress the updated tool with the commands:

gzip -N -d ecosconfig1314-linux.gz
chmod +x ecosconfig

Move the resulting ecosconfig executable into the tools/bin subdirectory of your eCos installation.

Other notes

Anonymous CVS access to the repository is read-only. If you would like to contribute a change, bug fix or other improvement to eCos then you should start by discussing it on the ecos-discuss mailing list. If applicable, you may wish to post (or put up for FTP) the relevant diff or patch.

You can obtain e-mail notifications when changes are made to the eCos repository. Refer to our keeping in touch page for further details.

Since the anonymous CVS repository is copied from the master repository, the author field and log entries are not accurate. Use the ChangeLog files at the top of each package hierarchy to obtain history on the sources in that package.

David Airlie of Parthus Technologies plc. has announced the availability of FTP snapshots of the complete eCos and RedBoot repository. It is possible to obtain complete snaphosts and individual files. This service may be particularly useful for people unable to run CVS directly, for example because of corporate firewalls.

Queries and problems regarding use of the anonymous CVS service should be directed to the ecos-discuss mailing list. The eCos development team, including the anonymous CVS maintainer, read this list. Please clarify that the mail concerns an anonymous CVS problem in the subject field.
